BEFORE YOU START
5 min setup

What You Need First

OpenClaw runs on something called Node.js. Think of Node.js as the engine ‚Äî OpenClaw is the car. You need the engine installed first.

What is Node.js? It's a program that lets JavaScript (the language websites use) run on your computer instead of just in a browser. OpenClaw is built with JavaScript, so it needs Node.js to work.

Step 1: Check if you already have Node.js

Open your terminal:

  • Mac: Press Cmd + Space, type "Terminal", press Enter
  • Windows: Press Windows key, type "Command Prompt" or "PowerShell", press Enter

Type this and press Enter:

node --version

If you see a number like v20.10.0 ‚Äî you're good! Skip to the next section.

If you see "command not found" or an error ‚Äî you need to install Node.js.

Step 2: Install Node.js

  1. Go to nodejs.org
  2. Click the big download button (it picks the right version for your computer automatically)
  3. Open the downloaded file and follow the installer steps
  4. When it finishes, close your terminal and open it again

Verify it worked:

node --version
You should see: A version number like v20.10.0 or higher
STEP 1
5 min

Install OpenClaw

Now that Node.js is ready, let's install OpenClaw.

What we're about to do: We're going to use npm (Node Package Manager) ‚Äî it comes with Node.js and is like an app store for developer tools. One command downloads and installs OpenClaw for you.

In your terminal, type this and press Enter:

npm install -g openclaw
This might take 1-2 minutes. You'll see a bunch of text scroll by. That's normal. Wait until you see your command prompt again (the line where you can type).

What that command does:

  • npm ‚Äî calls the Node package manager
  • install ‚Äî tells it to install something
  • -g ‚Äî means "global" (install it for your whole computer, not just one folder)
  • openclaw ‚Äî the name of the package to install

Verify it worked:

openclaw --version
You should see: A version number like 1.2.0 or higher
STEP 2
3 min

Start the OpenClaw Gateway

OpenClaw has two parts: the Gateway (runs in the background) and the Agent (what you talk to). Let's start the Gateway first.

What is the Gateway? Think of it like a server that runs on your machine. It handles all the behind-the-scenes work ‚Äî connecting to AI models, managing memory, running automations. Your agent can't work without it.

Start the Gateway:

openclaw gateway start

You should see something like:

‚úì Gateway started on http://localhost:3000
Important: The Gateway needs to keep running. Don't close this terminal window. You can minimize it, but leave it open in the background.

Open a NEW terminal window (Cmd+T on Mac, Ctrl+T on Windows) for the remaining steps.

Verify the Gateway is running:

openclaw status
You should see: Gateway: RUNNING
STEP 3
7 min

Set Up Your Workspace

OpenClaw needs a folder to store everything ‚Äî your agent's memory, configuration, skills, etc. Let's create that.

What is the workspace? It's a folder on your computer where OpenClaw stores all its files. Think of it as your agent's brain ‚Äî everything it knows and learns goes here.

Navigate to the workspace (OpenClaw creates it automatically):

cd ~/.openclaw/workspace

What that means:

  • cd ‚Äî "change directory" (like double-clicking a folder)
  • ~ ‚Äî your home folder
  • .openclaw ‚Äî a hidden folder (the dot makes it hidden) where OpenClaw lives
  • workspace ‚Äî the folder where your agent's files go

If you get an error that the folder doesn't exist:

mkdir -p ~/.openclaw/workspace
cd ~/.openclaw/workspace

Now let's create the three most important files:

File 1: IDENTITY.md

This tells your agent who it is.

Create the file:

nano IDENTITY.md

This opens a simple text editor in your terminal.

Type (or paste) this:

# Who You Are

- Name: InnerG
- Type: Personal AI Assistant  
- Purpose: Help with daily tasks, answer questions, automate workflows
- Voice: Direct, helpful, no fluff

To save and exit:

  • Press Ctrl + O then Enter to save
  • Press Ctrl + X to exit

File 2: USER.md

This tells your agent who YOU are.

Create the file:

nano USER.md

Type (or paste) this ‚Äî customize it for yourself:

# About Your Human

- Name: [Your Name]
- Location: [Your City]
- Timezone: [Your Timezone, e.g., America/New_York]
- What I do: [Your job/hobbies]
- How I like to communicate: [Short answers? Detailed explanations?]

Save and exit (Ctrl + O, Enter, Ctrl + X)

File 3: MEMORY.md

This is your agent's long-term memory. It starts empty and grows over time.

Create the file:

nano MEMORY.md

Type this:

# Long-Term Memory

Important things to remember go here.

## Projects
[Your agent will add things here as it learns]

## Preferences
[Your agent will remember what you like]

## Key Dates
[Important dates your agent should know]

Save and exit (Ctrl + O, Enter, Ctrl + X)

Congrats! Your agent now has an identity, knows who you are, and has a place to store memories.
STEP 4
3 min

Set Up Daily Memory

Besides long-term memory, your agent also keeps daily logs ‚Äî like a journal.

Why daily memory matters: Each day gets its own file. This lets your agent remember what happened on specific days, track progress over time, and stay organized. It's like having a daily notebook that your agent writes in.

Create the memory folder structure:

mkdir -p memory/episodic

What this does:

  • mkdir ‚Äî "make directory" (create folder)
  • -p ‚Äî create parent folders too if they don't exist
  • memory/episodic ‚Äî the folder path (memory folder, episodic subfolder)

Create today's log file:

touch memory/episodic/$(date +%Y-%m-%d).md

What that does:

  • touch ‚Äî creates an empty file
  • $(date +%Y-%m-%d) ‚Äî automatically inserts today's date (like 2026-02-28)
Your agent now has:
‚Ä¢ Long-term memory (MEMORY.md)
‚Ä¢ Daily logs (memory/episodic/)
‚Ä¢ Context about who you both are (IDENTITY.md + USER.md)
STEP 5
5 min

Start Chatting

Everything is set up. Time to talk to your agent.

Start a chat session:

openclaw chat

You'll see a prompt where you can type messages. Try these:

Test 1: Memory Check Type: What do you know about me?

Your agent should read USER.md and tell you what it knows. This proves the memory system is working.
Test 2: Teach It Something Type: Remember that I'm working on a project called InnerG Learn

Your agent should confirm it will remember this. Check MEMORY.md later ‚Äî it should be there.
Test 3: Ask for Help Type: Help me brainstorm 5 YouTube video ideas about AI automation

Your agent should generate ideas based on what it knows about you.

To exit the chat: Type exit or press Ctrl + C

Common Issues & Fixes

"command not found: node"

‚Üí Node.js isn't installed or wasn't added to your PATH. Reinstall from nodejs.org and restart your terminal.

"command not found: openclaw"

‚Üí OpenClaw didn't install correctly. Try npm install -g openclaw again. If that fails, you might need to use sudo npm install -g openclaw (Mac/Linux).

"Gateway won't start" or "Port 3000 in use"

‚Üí Something else is using port 3000. Find it with lsof -i :3000 (Mac/Linux) and kill it, or change OpenClaw's port in the config.

"Agent doesn't remember anything"

‚Üí Make sure MEMORY.md exists in your workspace folder. The agent needs that file to store long-term memories.

"Chat is really slow"

‚Üí Your agent might be using a slow AI model. Check your config to make sure you're using a faster model like GPT-4 or Claude.
